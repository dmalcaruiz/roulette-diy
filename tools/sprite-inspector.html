The atlas

One sheet, every part

The whole button lives in this 58×28 image: a drag handle and text-box tiles on the left, then five 9-slice layers — stroke, highlight, fill, shade, and shade-glint — packed side by side. Shown at 10× on a transparency checker so you can read individual pixels.

Transparent pixels are the checker; solid pixels are the art. The cyan block is the recolorable fill placeholder — at runtime it becomes the button's base color, with the blue and teal peek tones derived from it.

Palette

Five inks, two of them dynamic

Only the placeholder colors change at runtime. The fill drives the button's base; the shade and glint are recomputed from it in OKLCH so any button color keeps the artist's peek relationships. Stroke, highlight, and the checker never change.

Tiles

Each layer is a 9-slice frame

A layer draws four 6×6 corners and repeats 3-px edge tiles between them, so the button stretches to any width without distortion. Only the west side is stored — the east mirrors it. Below each layer's assembled frame are its unique source tiles with sheet coordinates.

Assembly

Stacked bottom to top

The peek (shade + glint) is laid first and anchored to the bottom, then the face — fill, highlight — sits on top, and finally the stroke wraps the whole silhouette so the outline is unbroken. The label is drawn last in the LoRes9 pixel face. Shown recolored to the app's PRIMARY.

Playground

Test a change

Recolor, resize, and press the button to see exactly what ships. Everything here runs the same composition the component does. Use it to pin down a change before you describe it.
